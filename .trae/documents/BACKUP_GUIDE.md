# 数据备份与恢复操作指南

> 本项目所有数据**全部本地存储**（IndexedDB 主存 + localStorage 轻量配置）。
> 建议**每周导出一次 JSON 备份**到电脑 / 网盘，避免手机丢失或损坏时数据无法找回。

---

## 1. 备份内容（v2 格式）

导出文件包含以下完整表：

| 表 | 路径 | 内容 |
|----|------|------|
| 账单 | `data.transactions` | 全部收支记录 |
| 账本 | `data.books` | 多账本列表 |
| 记账人 | `data.members` | 家庭成员 |
| 预算 | `data.budgets` | 日/月/年/全部/自定义预算 |
| 月度预算 | `data.monthBudgets` | 各月定制预算 |
| 自定义分类 | `data.customCategories` | 用户新增的分类 |
| 借条 | `data.lendBorrow` | 人情往来记录 |

> 主题、登录密码、最近备份时间等**轻量配置**不打包（已自动存于 localStorage，无需备份）。

### 文件结构示例

```json
{
  "version": 2,
  "exportedAt": "2026-06-17T10:30:00.000Z",
  "appName": "家庭记账本",
  "data": {
    "transactions": [ ... ],
    "books": [ ... ],
    "members": [ ... ],
    "budgets": { "day": 100, "month": 3000, "year": 36000, "all": 36000, "custom": 3000 },
    "monthBudgets": { "2026-06": 5000 },
    "customCategories": [ ... ],
    "lendBorrow": [ ... ]
  }
}
```

## 2. 导出备份

### 2.1 步骤

1. 打开 APP → 底部 Tab **我的** → **设置**
2. 找到 **数据备份** 入口
3. 点击「**导出完整备份文件**」
4. 浏览器下载 `记账备份_v2_YYYY-MM-DD_HHmm.json`
5. （APK 用户）文件会保存到手机下载目录

### 2.2 注意事项

- 文件大小一般 < 5MB（取决于账单量）
- 系统上限 50MB（超出拒绝）
- 导出后页面显示「最近备份时间」
- 文件名带时间戳，**不要改名**（避免误识别版本）

---

## 3. 导入恢复

### 3.1 步骤

1. 打开 APP → 设置 → **数据备份**
2. 点击「**选择文件恢复**」
3. 选择 `记账备份_v2_*.json`
4. 系统校验：
   - 文件大小 ≤ 50MB ✅
   - JSON 解析成功 ✅
   - `version === 2` ✅
   - 包含 `data.*` 节点 ✅
5. 弹窗「**确认恢复**」→ 二次确认（避免误操作）
6. 点击「**确认恢复**」→ 写入 IndexedDB → 提示恢复成功

### 3.2 行为说明

- **会覆盖**当前所有数据（不可撤销）
- 逐项写回（单项失败不影响其他）
- 恢复完成后**无需重启**，APP 内数据立即生效
- 提示信息：`账单 N 条 / 账本 M 个 / 借条 K 条`

### 3.3 异常情况

| 现象 | 原因 | 解决 |
|------|------|------|
| 「文件格式不正确：根节点不是对象」 | 不是 JSON | 重新导出 |
| 「文件版本不匹配」 | v1 旧版 | 用新版本 APP 重新导出 |
| 「文件过大」 | > 50MB | 拆分账本备份 / 删除老数据再导出 |
| 「文件读取失败」 | 文件损坏 | 重新导出 |
| 弹窗无法关闭 | 极少见，强制杀进程 | 重新打开后会自动恢复 |

## 4. 备份策略建议

| 用户 | 建议 |
|------|------|
| 普通用户 | 每周日导出一次到电脑 / 网盘 |
| 重度用户（>1000 笔） | 每 3 天导出一次 |
| 重要财务前 | 必导出 + 验证可恢复 |
| 多设备同步 | 设备 A 导出 → 设备 B 导入（账本 ID 会变，需重新设置账本） |

---

## 5. 备份的局限性

- **不包含**：登录密码、主题、APP 设置（这些在 localStorage，不建议跨设备）
- **账本 ID 会变**：恢复后的账本 ID 重新生成，关联数据通过 `bookId` 重新匹配
- **不包含**已删除的账单：仅备份当前内存中的活跃记录

## 6. Android 模拟器 / 真机操作

### 6.1 导出（APK）

```
设置 → 数据备份 → 导出完整备份文件
→ 系统弹窗「下载」→ 保存到 /Download/
```

### 6.2 导入（APK）

```
设置 → 数据备份 → 选择文件恢复
→ 系统文件选择器 → /Download/记账备份_v2_xxx.json
→ 二次确认 → 完成
```

### 6.3 adb 推送（高级）

```bash
# 把备份文件推到手机
adb push 记账备份_v2_xxx.json /sdcard/Download/

# 在 APP 中选择 /sdcard/Download/ 路径
```

## 7. 备份恢复与 IDB 关系

| 存储 | 内容 | 备份策略 |
|------|------|----------|
| IndexedDB（`fab_db`） | 账单/账本/分类/预算/借条 | **必须** JSON 备份 |
| localStorage | 主题/登录态/最近备份时间 | 不需备份（设备本地） |

> IndexedDB 数据**仅在本地**，不会上传任何服务器。删除 APP = 数据全清。备份是唯一安全网。

---

## 8. 故障排查

### 8.1 备份文件被微信/QQ 压缩损坏

→ 改用**邮件附件**或**云盘**分享，保留原始扩展名 `.json`

### 8.2 恢复后账单数对不上

→ 检查导入提示中的「账单 N 条」是否与备份文件一致
→ 如不符，文件可能在传输中损坏，重新导出

### 8.3 恢复失败但没有错误提示

→ 检查 `fab_db` 数据库是否被浏览器/系统清掉
→ 浏览器隐私模式不支持 IDB → 切回普通模式
